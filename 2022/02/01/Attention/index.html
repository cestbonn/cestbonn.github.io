Attention is all you need

Background

The fundamental hypothesis behind NLP is the belief in a universal space called "semantic". Any natural language can be transferred into space on a uniform scale. For the task of machine translation, the steps remained are to encode language A to the space, then decode the semantic representations back to language B.

RNN

However, there is no way for us the encode/decode sentences directedly as a whole. We need an efficient heuristic to achieve it one word by one word, which is the truth that we read and write from left to right. Therefore, RNN came up. It roughly accumulates (encode) the words relationships into the current context representation as current hidden state. The final hidden state \(h_l\) is expected to be the vector in "semantic space", representing the entire sentence, and thus target sentences can be generated by decoding \(h_l\) sequentially. Meanwhile, RNN has the drawback of precluding paralyzed training in batches for its inherently sequential nature.

LSTM

People find that not all words are important to the context representation so we should not simply accumulate hidden states. LSTM introduces cell states (\(c_{i-1})\) to control contributions of components of the context representation (\(h_i\)): current word (\(x_i\)) and previous context (\(h_{i-1}\)). Although LSTM was originally designed to solve the problem of information loss when decoding long sentences, it indicates that we should calculate the context representation in a selective way.

Attention

Attention is proposed to decode the final context representation selectively in terms of words in the translated sentence. To do it, we need weights between the current input of the decoder and all words in the translated sentence. This process is actually a query-key matching.
encoder: \(x_i\rightarrow \{k_i=h_i, v_i=h_i, h_i\}\)
decoder: \(x_j\rightarrow \{q_j=h_j, c_j, h_j\}\)
  • Weight (\(\alpha_{ij}\)) between \(x_i\) and \(x_j\) is, \[\alpha_{ij}=softmax(align(k_i, q_j))\]
  • Context representation (\(c_j\)) of words \(x_j\) is the weighted sum of \(v_i\), \[c_{j}=V^T\alpha_{:j}\ , \ V=[v_1, ..., v_m]\ ,\ \alpha_{:j}=[\alpha_{1j},...,\alpha_{mj}]^T\]
  • Predicted word (\(x_{j+1}\)) is induced from current hidden state or current context representation, \[x_{j+1}=softmax(FCNet(h_{j}\ or\ c_j))\]
  • Hidden state (\(s_{j+1}\)) is calculaed from the predicted current input, previous hidden state, and previous context representaion. \[h_{j+1}=RNNcell(x_{j+1}, h_{j}, c_{j})\] Now we have \(x_{j+1}\) and \(h_{j+1}\) to inducde \(q_{j+1}\) by repeat above steps. sequences of \(x_j\) is the translated result. Comparing the encoder and decoder, we treat hidden states as semantically consistent, but the context representation is word-specific. Also, the latter is only calculated in the decoder. It obeys the basic belief while making a trick about weights to help the network. paper

Self-attention

Previously, attention is only calculated when decoding, however, we can also generate weighted context representations while encoding, which is so-called self-attention. It works on one RNN instead of encoder-decoder structure, and is more flexible for downstream tasks. Intuitively, final context representation \(c_l\) now embeds context spacially, and the final hidden state \(h_l\) just simply accumulates context. So \(c_l\) is more powerful than \(h_l\) in original RNN for downstream tasks. Noticed that this self-attention is not the popular one, it in fact comes from the first self-attention related paper. In transformer, self-attention is re-designed systematically and is an independent block to produce \(c\).

Motivation

  1. In previous work, vectors of query, key and value are all based on RNN. However, RNN cannot be trained in paralle for its inherently sequential nature. So why not throw it way?

Design

Attention

encoder: \(x_i\rightarrow \{k_i=W_kx_i, v_i=W_vx_i\}\)
decoder: \(x_j\rightarrow \{q_j=W_Qx_j, c_j\}\)


Instead of align function, we use dot-product of query and key to estimate the relationships between \(x_j\) and \(x_i\), so that \[c_j=V\cdot softmax(K^Tq_j)\] \[x_{j+1}=softmax(FCNet(c_j))\]

Self-attention

In this part, \(j\) is the same as \(i\).
encoder: \(x_i\rightarrow \{k_i=W_kx_i, v_i=W_vx_i, q_i=W_Qx_i\}\)

\[c_j=V\cdot softmax(K^Tq_j)\]

Multi-head Attention/Self-attention

Just use multiple independent attention/self-atttention to produce \(c\), then concatenate them.

Trick

  • Scale \(QK^T\) by \(1/\sqrt{d_k}\). Large \(QK^T\) will push softmax function into regions where it has extremely small gradients.
